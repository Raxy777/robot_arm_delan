# Running this project on Google Colab

You have no local GPU, so training happens on Colab. MuJoCo runs headless there,
so **data generation and training both happen in the notebook** — no local
MuJoCo needed at all. (The MLP is small enough that even the CPU runtime trains
it in seconds; pick a GPU runtime mainly for Week 9's DeLaN.)

The code lives in a GitHub repo; the notebook clones it each session.

---

## One-time: push the project to GitHub

From the `robot_arm_delan/` folder on your machine:

```bash
git init
git add .
git commit -m "Robot arm DeLaN project: Weeks 7-8"

# create an EMPTY repo on github.com first (no README), then:
git remote add origin https://github.com/<your-username>/robot_arm_delan.git
git branch -M main
git push -u origin main
```

`.gitignore` already excludes `data/`, `models/`, and `outputs_run/` — those are
regenerated by the scripts, so the repo stays small and clean.

> Public vs private: a public repo is simplest (the notebook clones with no
> auth) and doubles as a portfolio link for placements. If you keep it private,
> see the auth note at the bottom.

---

## Each session: open the notebook in Colab

1. Go to [colab.research.google.com](https://colab.research.google.com) →
   **File → Upload notebook** → pick `colab/week8_train.ipynb`
   (or **File → Open notebook → GitHub** and paste your repo URL once it's pushed).
2. *(optional)* **Runtime → Change runtime type → T4 GPU.** Not needed for the
   Week-8 MLP; worth it for Week 9.
3. **Runtime → Run all.**

The notebook: clones your repo → installs `mujoco`+`torch` → generates the
datasets → trains the MLP → prints the in-dist vs OOD tables → offers the
trained `mlp.pt` for download.

---

## Iterating (you changed a `.py` and want it on Colab)

```bash
# locally
git add -u && git commit -m "tweak excitation" && git push
```

Then in Colab just re-run the clone cell (it does `git pull` if the repo is
already there). No re-upload, no lost work.

---

## Getting trained models back

The last cell calls `files.download('models/mlp.pt')`. Drop that file into your
local `robot_arm_delan/models/` and `evaluate_model.py --backend analytic` (or
`mujoco`) runs the closed-loop benchmark locally with the Colab-trained weights.

Alternatively mount Drive (`from google.colab import drive`) and save straight
there — handy once you have several week's models.

---

## Private repo auth (skip if public)

Colab can't see a private repo without a token. Simplest path: create a
fine-grained GitHub Personal Access Token (read-only, that one repo), then in
the clone cell use:

```python
import os
os.environ["GH_TOKEN"] = "github_pat_..."   # paste, don't commit
!git clone https://$GH_TOKEN@github.com/<user>/robot_arm_delan.git
```

Better yet, store it via Colab's **🔑 Secrets** panel and read it with
`google.colab.userdata.get('GH_TOKEN')` so the token never sits in a cell.
